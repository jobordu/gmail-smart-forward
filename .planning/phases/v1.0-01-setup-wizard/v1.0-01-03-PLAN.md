Create `scripts/setup.js` as a CommonJS script. It MUST:

1. **Verify `@inquirer/prompts` CJS compatibility** before writing any wizard code:
   ```bash
   node -e "const i = require('@inquirer/prompts'); console.log(typeof i.input)"
   ```
   - If this errors with `ERR_REQUIRE_ESM`, install using dynamic import shim:
     `npm install --save-dev @inquirer/prompts` then use `import('@inquirer/prompts')` wrapped in an async function.
   - If it succeeds (outputs `function`), use `require('@inquirer/prompts')`.

2. **Install dependencies** if not already installed:
   ```bash
   npm install --save-dev @inquirer/prompts chalk
   ```
   Use `chalk@4.x` (NOT 5.x — ESM only):
   ```bash
   npm install --save-dev chalk@4
   ```

3. **Implement the wizard flow** in `scripts/setup.js`:

```
Prompt 1 — Forward-to email (FORWARD_TO_EMAIL):
  message: "Forward matching emails to (email address):"
  default: existing .env value if present
  validate: isValidEmail() — reject with "Invalid email format — enter a valid address like user@example.com"

Prompt 2 — Allowed senders (ALLOWED_SENDERS):
  message: "Allowed sender addresses (comma-separated, leave blank to allow all):"
  default: existing .env value if present
  validate: isValidEmailList() — reject with "One or more email addresses are invalid"

Prompt 3 — Allowed domains (ALLOWED_DOMAINS):
  message: "Allowed sender domains (comma-separated, e.g. supplier.com — leave blank to skip):"
  default: existing .env value if present
  (no format validation — domains are free-form)

Prompt 4 — Languages (for keyword generation):
  type: checkbox (multi-select)
  message: "Which languages do your emails arrive in? (space to select, enter to confirm)"
  choices: en (English), fr (French), es (Spanish), pt (Portuguese), de (German), zh (Chinese), ja (Japanese)
  default: pre-select 'en' if no existing keywords, or skip if SUBJECT_KEYWORDS already set

Prompt 5 — File types (ATTACHMENT_EXTENSIONS):
  message: "Allowed attachment file types (comma-separated, e.g. pdf,xlsx):"
  default: existing value or "pdf"
  validate: isValidFileTypeList() — reject with "Enter file extensions without dots (e.g. pdf, not .pdf)"

Prompt 6 — Confirmation summary:
  Print a summary block using chalk before asking:
    chalk.bold('\n── Values to write to .env ──')
    chalk.cyan('Forward to:') + value
    chalk.cyan('Allowed senders:') + value or '(all senders)'
    chalk.cyan('Allowed domains:') + value or '(none)'
    chalk.cyan('File types:') + value
    chalk.cyan('Subject keywords:') + generated keywords (comma count)
    chalk.cyan('Attachment keywords:') + generated keywords
    chalk.bold('──────────────────────────────')
  Then: confirm prompt — "Write these values to .env now?"
  If user answers No (or Ctrl+C): print "Aborted — .env not changed." and exit 0.

After confirmation:
  Call keyword-gen.generate(languages) to get SUBJECT_KEYWORDS + ATTACHMENT_FILENAME_KEYWORDS.
  Build the updates object with ALL required keys:
    FORWARD_TO_EMAIL, ALLOWED_SENDERS, ALLOWED_DOMAINS,
    ATTACHMENT_EXTENSIONS, SUBJECT_KEYWORDS, ATTACHMENT_FILENAME_KEYWORDS,
    DRY_RUN (set to 'true' if not already in .env, preserve if exists),
    ENABLE_LIVE_FORWARDING (set to 'false' if not already in .env, preserve if exists)
  Call env-parser.write(updates).
  Print: chalk.green('✓ .env written successfully.')
  Print: 'Next step: run npm run push, then run bootstrapProperties() in the Apps Script editor.'
```

**Error handling:**
- Wrap the entire `main()` in try/catch. On unhandled error: print `chalk.red('Setup failed: ' + err.message)` and `process.exit(1)`.
- If `@inquirer/prompts` uses dynamic `import()`, wrap in an async IIFE.

**Ctrl+C handling:**
Inquirer handles SIGINT by throwing `ExitPromptError`. Catch it and print "Aborted — .env not changed." then exit 0.
